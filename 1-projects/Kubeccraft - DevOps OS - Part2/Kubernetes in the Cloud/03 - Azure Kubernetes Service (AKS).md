All providers has Kubernetes managed service. AKS in case of Azure.

Mischa's approach when learning/trying a new service is to create it through Azure Portal first. Only then create the Terraform (or other) code.


## Create Kubernetes cluster via Azure Portal

### NodePool

_System_ - Database operator, External Secrets operator (infrastructure of the cluster)

_User_ - User workflows

### Pricing tier

- _Free_ - management is free, but charged for VM, storage, and network usage
- _Standard_ - for mission-critical and production workloads
- _Premium_ - Standard with 2 years of support

Kubernetes version - `1.33.6` (default) vs homelab `1.33.3`:

### Authentication and Authorization

1. Local accounts with Kubernetes RBAC - anyone with `~/.kube/config` file has acces
2. MS Entra ID with Kubernetes RBAC - needs MS ID to access the cluster
3. MS Entra ID only Azure RBAC
